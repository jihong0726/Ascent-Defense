# 🚀 Ascent Defense: 策略点击塔防原型

## ✨ 项目简介

**Ascent Defense** 是一个基于 **塔防 (Tower Defense)** 和 **Rogue-lite 变异升级** 机制的策略游戏原型。

本项目旨在探索**最小化的界面交互**下，如何实现**最大化的策略深度**。玩家在 4x4 网格上部署和升级防御塔，并在每一波结算后，通过选择永久性的**科技变异 (Mutations)** 来应对日益强大的敌人。

### 🎮 核心玩法机制

*   **部署与升级:** 在 4x4 网格地图的空地上建造和升级防御塔（T1, T2）。
*   **资源管理:** 管理 **GOLD**（用于建造/升级）和 **TP (Technology Points)**（用于选择变异）。
*   **变异系统:** 每次清理完一波敌人后，玩家获得 TP，用于选择强大的、**不可逆转的** Rogue-lite 式全局升级。
*   **核心防御:** 目标是保护核心 (`C3`) 不被敌人摧毁。

## 🛠️ 如何运行项目

本项目是纯前端实现，只需将以下三个文件放在同一目录下，并用浏览器打开 `index.html` 即可。

1.  `index.html`
2.  `style.css`
3.  `game_ui.js`

### 文件说明

| 文件名 | 职责 | 备注 |
| :--- | :--- | :--- |
| `index.html` | 页面结构 | 基础布局和 UI 元素。 |
| `style.css` | 视觉样式 | 项目的颜色和网格布局。 |
| `game_ui.js` | **核心逻辑** | 包含所有游戏状态、点击事件处理、建造/升级/出售、回合结算和变异逻辑。**所有交互修复均在此文件。** |

### 已知限制 (待开发)

*   **战斗深度：** 塔攻击目前仅针对**队首敌人**，未实现射程 (`range`) 限制或寻路优化。
*   **敌人模型：** 敌人模型简化，缺乏护甲、抗性和特殊技能。
*   **路径系统：** 使用简化的硬编码路径，未实现动态 A\* 寻路来应对玩家的堵路行为。
